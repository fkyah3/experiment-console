# ExperimentConsole 使用说明

## 启动方式

在 Godot 编辑器里：
1. 左侧 FileSystem 找到 `ExperimentPanel.tscn`
2. 右键 → **Run as Main Scene**（或选中后 `Shift + F6`）

## 第一次使用

点 `⚙ 设置`，填入 DeepSeek API Key，点保存。Key 存在 `config.cfg`，启动时自动加载，只需要输一次。

---

## 操作流程

### 1. 搭积木

点击 `+ 添加积木` 选择角色类型：

| 积木类型 | 用途 |
|:---------|:------|
| system | 系统提示词（语言锚定、角色设定） |
| user | 用户提问 |
| assistant (content) | AI 的文本回复（可手动填入用于多轮） |
| assistant (reasoning) | AI 的思考过程（手动填入 reasoning_content） |
| assistant (tool_calls) | AI 请求调工具 |
| tool | 工具执行结果 |

**每块积木的操作：**
- `▼` 展开查看/编辑完整内容
- `✏` 弹窗编辑所有字段（role / content / reasoning）
- `↑ ↓` 调整顺序
- `✕` 删除

**展开后分为两个区域：**
- `━━━ reasoning_content（原始思考过程）━━━` — 橙色标签，API 返回的原始推理过程，可编辑
- `━━━ content（最终回答）━━━` — 蓝色标签，最终文本

折叠行如果有 reasoning 内容会在末尾显示 `🧠<字符数>` 标签。

### 2. 设参数

顶部参数面板：

| 参数 | 说明 |
|:-----|:------|
| 模型 | `deepseek-v4-flash`（默认，经济快捷）或 `deepseek-v4-pro`（最强性能） |
| 思考模式 | `enabled` — 输出 `reasoning_content`（思维链）；`disabled` — 纯文本，不思考 |
| 推理强度 | `high` / `max`。`max` 用于复杂 Agent 场景，普通提问用 `high` |
| max_tokens | 最大输出长度（默认 4096，上限 384000） |
| 温度 | 0.0~2.0，默认 0.0（确定性输出）。**思考模式下不生效** |
| 实验标题 | 存文件时用的文件名前缀，不填则取第一条消息 |

**重要：** 思考模式（enabled）下，temperature、top_p、presence_penalty、frequency_penalty 四个参数设置不会报错，但**不生效**。要控制随机性只能用模型本身的确定性。如果想调温度，需要关闭思考模式（disabled）。

### 3. 使用模板

点 `📋 模板`：

**内置模板：**
- **中文锚定测试**：system(锚定指令) + user(中文提问)
- **前置条件测试**：user("我表妹叫我奶奶什么？")
- **自定义（空）**：空 messages，自己搭

**保存当前为模板：**
点 `📋 模板` → 选底部的 `💾 保存当前为模板` → 输入名称 → 保存

模板是**插入式**的——只追加消息到当前列表末尾，不动参数面板的设置。

### 4. 发送

点 `▶ 发送`：
- 消息列表末尾自动追加一条空的 assistant
- 流式内容逐字追加，期间实时更新预览
- 完成后日志区显示响应体 + token 用量（含 reasoning_tokens）

### 5. 存文件

点 `💾 保存到文件`，存到 `experiments/` 目录下。

保存的内容：
- 原始积木 JSON（reasoning 和 content 分离）
- API 格式 JSON（reasoning 拼入 content）
- 完整请求体 JSON
- 完整响应体 JSON（含原始 reasoning_content）
- usage 统计（prompt / completion / reasoning tokens）
- 实验备注

### 6. 看日志

- 默认显示请求体
- 切换"响应体"标签看返回
- 点 `查看请求体` 大窗口查看
- usage 行显示 prompt / completion / reasoning 的 token 数

### 7. 设置

点 `⚙ 设置`：

| 设置项 | 说明 |
|:-------|:------|
| DeepSeek API Key | 密码模式输入，保存后即时生效 |
| 实验存储路径 | 默认 `user://experiments/`，可改，点"打开文件夹"定位 |
| 模板存储路径 | 默认 `user://templates/`，可改，点"打开文件夹"定位 |
| 会话存储路径 | 默认 `user://sessions/`，可改，点"打开文件夹"定位 |

**打开文件夹**按钮会用系统文件管理器打开对应目录。

## API 参数参考

### 基本信息

| 参数 | 值 |
|:-----|:----|
| 端点 | `https://api.deepseek.com/chat/completions` |
| 认证 | `Authorization: Bearer sk-...` |
| 上下文长度 | 1M tokens |
| 输出上限 | 384K tokens |

### 模型选择

| 模型名 | 说明 | 价格（输入/输出，百万 tokens） |
|:-------|:-----|:------------------------------|
| `deepseek-v4-flash` | 推荐默认，经济快捷 | 缓存命中 ¥0.02 / ¥2 |
| `deepseek-v4-pro` | 最强性能，当前 2.5 折 | 缓存命中 ¥0.1 / ¥24（优惠至 2026-05-31） |

注意：`deepseek-chat` 和 `deepseek-reasoner` 是旧别名，**2026-07-24 弃用**，请直接用新名称。

### 流式输出

流式模式下，chunk 格式如下：

```
# reasoning_content chunk
{"choices":[{"index":0,"delta":{"reasoning_content":"思考过程..."},"finish_reason":null}]}

# content chunk
{"choices":[{"index":0,"delta":{"content":"最终回答..."},"finish_reason":null}]}

# 结束
{"choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}

# usage 信息
{"choices":[],"usage":{"prompt_tokens":100,"completion_tokens":50,"total_tokens":150}}

# 结束标记
data: [DONE]
```

- `delta.reasoning_content` 和 `delta.content` **不会同时出现**，需分别累积
- 请求排队时会收到 `: keep-alive` 注释行，解析时需忽略

## 错误排查

| 错误 | 原因 | 处理 |
|:-----|:-----|:------|
| 连接错误 | 网络不通 / API key 无效 | 检查设置中的 Key |
| 400 | 请求体格式错误（常见于工具调用时 reasoning 未回传） | 检查 assistant 积木是否缺少 reasoning 字段 |
| 401 | 认证失败 | API key 错误或过期 |
| 402 | 余额不足 | 去 DeepSeek 平台充值 |
| 429 | 请求频率过高 | 等待后重试 |
| 500/503 | 服务器繁忙 | 稍后重试 |

## 多轮对话怎么做

实验台是**积木式**的，不自动保留上下文。要模拟多轮对话：

```
第一轮：搭积木 [system] + [user]  → 发送
        收到 assistant，留着别删

第二轮：第一轮的 [system] + [user] + [assistant] 保留
        再加一条 [user] "接着说"
        发送

第三轮：以上全部保留 + 再加 [user]
```

每轮都需要你自己决定 messages 的组成——删掉某一块就是新一轮独立实验，留着就是延续对话。

## 工具调用怎么做

1. 搭积木 `[user]` 例如"北京的天气怎么样？"
2. 点发送，模型返回 `assistant(tool_calls)`
3. 手动添加 `[tool]` 积木，填入工具执行结果（如"多云 7~13°C"）
4. 可追加 `[user]` 继续对话
5. 再点发送

**关键：** 有工具调用的轮次，上一轮的 `reasoning_content` **必须保留**在 assistant 积木的 reasoning 字段中，否则 API 返回 400。

## 文件存储

```
experiments/              ← 实验记录 .md 文件
templates/                ← 模板 .json 文件
sessions/                 ← 主场景的 .md 会话文件
config.cfg                ← 配置文件（API key、路径）
docs/deepseek-api/        ← DeepSeek API 参考文档
```

Windows 实际路径：`%APPDATA%\Godot\app_userdata\ExperimentConsole\`

每个 .md 实验文件包含：
- YAML frontmatter（参数快照：模型、思考模式、强度、温度等）
- 原始积木 JSON（reasoning 和 content 分离）
- API 格式 JSON（发送到服务器的实际内容）
- 请求体 JSON
- 响应体 JSON（含原始 reasoning_content）
- usage JSON（含 reasoning_tokens 统计）
- 实验备注
