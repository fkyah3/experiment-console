# 实验台使用说明

## 启动方式

在 Godot 编辑器里：
1. 左侧 FileSystem 找到 `test/ExperimentPanel.tscn`
2. 右键 → **Run as Main Scene**（或选中后 `Shift + F6`）

想切回主场景，同样的方法选 `Scenes/Main/MainScene.tscn`。

## 第一次使用

点 `⚙ 设置`，填入 DeepSeek API Key，点保存。Key 存在 `user://config.cfg`，启动时自动加载，只需要输一次。

---

## 操作流程

### 1. 搭积木

点击 `+ 添加积木` 选择角色类型。

| 积木类型 | 用途 |
|:---------|:------|
| system | 系统提示词（语言锚定、角色设定） |
| user | 用户提问 |
| assistant (content) | AI 的文本回复（可手动填入用于多轮） |
| assistant (reasoning) | AI 的思考过程（手动填入 reasoning_content） |
| assistant (tool_calls) | AI 请求调工具 |
| tool | 工具执行结果 |

**每块积木的操作：**
- `▼` 展开查看/编辑完整内容
- `✏` 弹窗编辑所有字段（role / content / reasoning）
- `↑ ↓` 调整顺序
- `✕` 删除

**展开后分为两个区域：**
- `━━━ reasoning_content（原始思考过程）━━━` — 橙色标签，API 返回的原始推理过程，可编辑
- `━━━ content（最终回答）━━━` — 蓝色标签，最终文本

折叠行如果有 reasoning 内容会在末尾显示 `🧠<字符数>` 标签。

### 2. 设参数

顶部参数面板：
- **模型**：默认 deepseek-v4-flash（快/便宜），也可选 deepseek-v4-pro（强/贵）
- **思考模式**：enabled（输出 reasoning_content）/ disabled（纯文本，不思考）
- **推理强度**：high / max / 不传
- **max_tokens**：最大输出长度（默认 4096）
- **实验标题**：存文件时用的文件名前缀，不填则取第一条消息

### 3. 使用模板

点 `📋 模板`：

**内置模板：**
- **中文锚定测试**：system(锚定指令) + user(中文提问)
- **前置条件测试**：user("我表妹叫我奶奶什么？")
- **自定义（空）**：空 messages，自己搭

**保存当前为模板：**
点 `📋 模板` → 选底部的 `💾 保存当前为模板` → 输入名称 → 保存

模板是**插入式**的——只追加消息到当前列表末尾，不动参数面板的设置。

### 4. 发送

点 `▶ 发送`：
- 消息列表末尾自动追加一条空的 assistant
- 流式内容逐字追加，期间实时更新预览
- 完成后日志区显示响应体 + token 用量（含 reasoning_tokens）

### 5. 存文件

点 `💾 保存到文件`，存到 `user://experiments/` 目录下。

保存的内容：
- 完整请求体 JSON
- 完整响应体 JSON（含 reasoning_content）
- usage 统计（prompt / completion / reasoning tokens）
- 实验备注

### 6. 看日志

- 默认显示请求体
- 切换"响应体"标签看返回
- 点 `查看请求体` 大窗口查看
- usage 行显示 prompt / completion / reasoning 的 token 数

### 7. 设置

点 `⚙ 设置`：

| 设置项 | 说明 |
|:-------|:------|
| DeepSeek API Key | 密码模式输入，保存后即时生效 |
| 实验存储路径 | 默认 `user://experiments/`，可改，点"打开文件夹"定位 |
| 模板存储路径 | 默认 `user://templates/`，可改，点"打开文件夹"定位 |
| 会话存储路径 | 默认 `user://sessions/`，可改，点"打开文件夹"定位 |

**打开文件夹**按钮会用系统文件管理器打开对应目录。

## 文件存储

```
user://experiments/         ← 实验记录 .md 文件
user://templates/           ← 模板 .json 文件
user://sessions/            ← 主场景的 .md 会话文件
user://config.cfg           ← 配置文件（API key、路径）
```

Windows 实际路径：`%APPDATA%\Godot\app_userdata\MountainShift\`

每个 .md 实验文件包含：
- YAML frontmatter（参数快照）
- messages JSON
- request JSON
- response JSON（含原始 reasoning_content）
- usage JSON
- 实验备注
