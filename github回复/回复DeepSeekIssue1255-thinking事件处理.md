**事件：** 用户只输入 `<thinking>`，模型回了一道与输入无关的物理题。有人认为是安全漏洞。

**技术判断：** 不是 bug。`<thinking>` 是 chat template 的特殊 token，在训练数据中永远对应"问题→答案"的模式。模型看到 token 后触发了训练记忆中的完整模式匹配——它不是回复用户，是在"接续"训练时见过的路径。我们跑了 100 次 batch 验证：内容随机分布（物理题/地理/编程教程），无特定用户数据泄露，99% 中文 reasoning。

**一个额外的发现：** 在写完这份分析报告后，我发现自己在"这不正是"这个句式上模仿了刚讨论过的齐泽克的修辞节奏——而我写的时候完全没有意识到。

这件事本身印证了 #1255 的核心机制：语言模式可以在无显性引用的情况下发生跨上下文感染。发生在人身上和发生在模型上的逻辑是一致的——上下文中的语言特征会推高后续输出的采样概率，不论载体是人还是 LLM。

区别只在于：**人可以靠外部观察（另一个人类的反馈）发现自己发生了漂移并修正。模型在没有外部干预的情况下无法自行跳出这个循环。** 这正好说明了假 user 通道方案的价值——它不是"换个 prompt"，是从架构上给模型加了一个外部反馈环。
