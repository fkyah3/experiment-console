## 三、根因分析

### 与第一个 session 的本质差异

| 维度 | Session 1（工程层） | Session 2（本 session） |
|:-----|:-------------------|:-----------------------|
| 漂移层 | reasoning | text |
| 漂移形式 | 中文→英文（语言切换） | 简体→繁体（字符级） |
| 触发器 | 工具返回英文代码 | 纯对话，无工具调用 |
| 污染源 | 英文 token 占比超阈值 | 训练数据中简繁混合分布 |
| 是否可逆 | 99.4% 不可逆 | 用户指出后 AI 立即承认 |
| Reasoning 中文率 | 0.58% | 28.4% |

### 本次漂移的根因：**训练数据的简繁混合分布**

1. **没有外部污染源**：无工具调用，无英文代码注入
2. **话题特征**："常熟阿诺"是中国互联网梗，相关语料在训练数据中可能有大量来自港台/海外中文社区的讨论（繁体字为主）
3. **字符级泄漏**：模型在 token 采样时，上下文激活了繁体字的概率分布，导致部分常见字（为/说/时/个/会）以繁体形式输出
4. **自纠正**：用户指出后，AI 的下一步 text 立即回到简体（§4827° 的回复只有少数几个繁体字作为示例列出）

### 总结式对比

```
Session 1（工程层）:
  read 英文代码 → 英文 token 洪水 → reasoning 切英文 → 自锁不可逆
  
Session 2（中控层）:
  闲聊"常熟阿诺" → 训练数据中相关语料含繁体 → 字符级繁体泄漏 → 用户指出后恢复
```

## 四、对移山项目的意义

### 新发现

1. **有两种不同性质的漂移**：
   - **语言切换型**（zh→en）：由工具返回内容触发，reasoning 层，不可逆
   - **字符泄漏型**（简体→繁体）：由训练数据分布触发，text 层，可自纠正

2. **字符泄漏型无需特殊处理**：AI 自己能识别并纠正，影响范围极小（本次仅 5 个字符，且仅发生一次）

3. **Reasoning 中文率 28.4% 远高于 Session 1 的 0.58%**：说明 system prompt 的强度是关键变量

### 对移山方案的建议优先级

| 优先级 | 问题 | 对策 |
|:------|:-----|:-----|
| P0 | reasoning 切英文（Session 1 类型） | 需要方案 |
| P3 | text 零星繁体（本 Session 类型） | 可忽略，AI 自纠正 |
