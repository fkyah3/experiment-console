## 工程层简报：推理语言漂移复现实验

### 做了什么

***

**目标：** 确认"工具返回英文代码 → reasoning 语言从中文切换到英文"是否可复现，找到最小复现条件。

### 实验过程

| 轮次 | 方案                              | prompt          | 注入方式         | 代码量     | 结果       |
| :- | :------------------------------ | :-------------- | :----------- | :------ | :------- |
| 1  | baseline-完整复现 (模板)              | 我们的             | tool→user 旁路 | 51KB 分批 | ❌ zh     |
| 2  | min\_experiment.py (Python)     | 我们的             | user 旁路      | 5.7KB   | ❌ 三轮全 zh |
| 3  | V7-真实provider.ts (模板)           | 我们的             | tool→user 旁路 | 52KB    | ❌ zh     |
| 4  | min\_experiment\_v3.py (Python) | **opencode 原版** | user 旁路      | 3.2KB   | ❌ 三轮全 zh |

### 涉及文件

| 文件                                                              | 作用                                 |
| :-------------------------------------------------------------- | :--------------------------------- |
| `templates/baseline-完整复现.json` \~ `V7-真实provider.ts(52KB).json` | 静态模板，注入 52KB 真实代码                  |
| `scripts/APIBuilder.gd`                                         | tool→user 旁路转换                     |
| `scripts/DeepSeekStreamClient.gd`                               | 错误可见性加强                            |
| `doc/移山/分析层_数据库研究/min_experiment.py`                            | 分析层交付的多轮 Python 脚本                 |
| `doc/移山/分析层_数据库研究/min_experiment_v3.py`                         | 工程层追加：opencode 真实 prompt + user 旁路 |

### 结论

**静态模板无法复现漂移，与 prompt、注入方式、注入量均无关。** 真实 opencode 场景中，"模型自己产出 reasoning → 被带回下一轮 → 看到更多英文 → reasoning 变化"的自锁环在静态模板中不存在。单次 52KB 代码注入只是"一条很长的历史消息"，不是"模型必须带回下轮的自身思考"。

### 下一步

**V2 多轮对话支持。** 核心能力：发送 → 响应自动追加到消息列表 → 推理结果作为 `reasoning_content` 带入下一轮 → 逐轮累积英文冲击。有了这个，7 套阈值模板才能从"一次性静态注入"改为"逐轮渐进式传导"——那才是真实漂移的发生条件。
