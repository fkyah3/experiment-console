# 社区验证 04 — #1262：thinking/non-thinking 模式切换断裂

> 2026-05-15
> Issue 链接：https://github.com/deepseek-ai/DeepSeek-V3/issues/1262
> 报告人：weizhang-dotcom（同 #1304）

---

## 现象

同一个对话中切换 thinking（思考）和 non-thinking（不思考）模式后，模型在工具调用场景下的推理出现断裂。具体表现为：模式切换后模型似乎丢失了之前的推理上下文，行为不一致。

## 移山验证

该问题与 #1255 的根因直接关联。V4 的 Interleaved Thinking 机制将 reasoning 历史改为永久保留。当模式切换时：
- thinking 模式生成的 reasoning_content 在 non-thinking 模式下如何处理
- non-thinking 模式没有 reasoning 字段，之前积累的推理链如何衔接

API 文档中缺少对模式切换时 reasoning 处理方式的明确说明。这导致开发者在实现"同一次对话中切换 thinking 模式"时只能自己猜规则。

## 对移山的参考价值

移山的假 user 通道方案从架构上绕过了这个问题：主 agent 始终在 thinking 模式下运行，工具调度由 non-thinking 的子 agent 在独立上下文中完成。模式切换不发生在同一个上下文中，不会出现上下文丢失问题。

该 Issue 进一步支撑了移山"上下文隔离"的设计方向——问题不是 thinking 模式切换不好，而是同一个上下文中切换模式没有可靠的上下文衔接机制。
