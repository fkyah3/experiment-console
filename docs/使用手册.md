## 一、基本概念

### 积木（Message Block）

积木是实验台最小的构建单元。一个积木代表一条 AI 对话消息。多个积木按顺序组成一次对话。

### 积木类型

| 积木类型 | role 值 | 什么时候用 |
|:---------|:--------|:----------|
| **system** | `system` | 设定 AI 的行为规则、角色、输出格式。**system 内容不能为空。** |
| **user** | `user` | 用户的问题或请求。 |
| **assistant (content)** | `assistant` | AI 的最终回答。手动填写时可用于预设示例。 |
| **assistant (reasoning)** | `assistant` | AI 的思考过程。手动填写时 system prompt 要求 model 输出。 |
| **assistant (tool_calls)** | `assistant` | AI 发起的工具调用请求。用于 function calling 场景。 |
| **tool** | `tool` | 工具调用的返回结果。对应 tool_call_id。 |

### 积木编辑

- **展开/收起**：点积木行右侧 `▼`/`▲` 展开 content 和 reasoning 的编辑框
- **编辑**：点 `✏` 打开完整编辑弹窗（role / content / reasoning 均可改）
- **排序**：`↑` 上移、`↓` 下移
- **删除**：`✕`
- **预览**：每一行显示 content 前 50 字；有 reasoning 时显示 `🧠[字数]`

## 二、参数说明

| 参数 | 范围 | 含义 |
|:-----|:------|:------|
| **模型** | `deepseek-v4-flash` / `deepseek-v4-pro` | 使用的 DeepSeek 模型版本。Flash 速度快，Pro 质量高。 |
| **参数模式** | `官方预设` / `自定义` | 控制参数权限。**官方预设**使用 DeepSeek 推荐配置（temperature/top_p/freq_penalty 由 API 管理，设置不生效）。**自定义**解锁全部参数控制权。

**注意：这不控制模型是否思考。** 模型内部始终有轻量推理，`官方预设` 仅输出完整推理链，`自定义` 仅输出极短推理或不输出。真正的区别在于你能否控制温度/top_p/freq_penalty。 |
| **推理强度** | `high` / `max` / `不传` | 仅 `官方预设` 模式下生效。控制推理深度。high 适合大多数场景，max 用于复杂推理。 |
| **max_tokens** | 256 – 384000 | 模型单次回答的最大 token 数（含 reasoning）。数字越大回答越长。 |
| **温度** | 0.0 – 2.0 | 仅 `自定义` 模式下生效。回答的随机性。0.0 最确定（每次都一样），越高越随机。 |
| **top_p** | 0.0 – 1.0 | 仅 `自定义` 模式下生效。核采样，只从累积概率达到 p 的 token 里选。通常与温度二选一。 |
| **freq_penalty** | -2.0 – 2.0 | 仅 `自定义` 模式下生效。频率惩罚，正值让模型少重复，负值鼓励重复。 |
| **实验标题** | 自由文本 | 保存实验时生成的 .md 文件名标题。留空自动取第一条 user 消息的前 20 字。 |

## 三、批处理实验

### 验证脚本

批量运行时可在弹窗中填写 Python 验证脚本。验证脚本格式要求：

```python
# 验证脚本拼接规则：AI 生成的代码在前，验证脚本在后
# 验证脚本中 print("PASS") 代表通过，print("FAIL: 原因") 代表不通过
# exit code 0 = 通过，非 0 = 失败

# 示例：验证反转链表函数
assert reverse_list([1, 2, 3]) == [3, 2, 1]
print("PASS")
```

验证结果会追加到 `_summary.md` 的每轮明细表和新增的 `验证结果` 段。

### batch 目录结构

```
experiments/
└── batch_20260507_0304_归真/
    ├── _summary.md                  ← 批次汇总（统计 + 每轮明细）
    ├── 20260507_0304_batch_001.md   ← 第 1 轮
    ├── 20260507_0304_batch_002.md   ← 第 2 轮
    └── ...                          ← 最多 100 轮
```

## 四、浏览报告

每个批次目录可通过实验台的 `📖 浏览报告` 按钮进入 BatchReader 查看。

### 左侧面板

| 区域 | 指标 | 含义 |
|:-----|:-----|:------|
| **参数** | 模型/思考/强度 | 该批次的固定参数配置 |
| | 总计/成功/失败 | 总轮数、成功失败数 |
| **统计** | rt最小值/最大值/平均值/中位数 | 该批次所有成功轮次的 reasoning_tokens 分布 |
| | 波动倍数 | rt最大值 / rt最小值，波动越小表示推理 token 越稳定 |
| | prompt_avg / completion_avg / total_avg | 各类 token 的平均消耗 |
| | 中文占比/英文占比 | reasoning 语言比例 |
| **耗时速度** | 总用时/平均每轮 | 整个批次的总耗时和每轮平均耗时 |
| | 总 tokens / 平均 tok/s | token 消耗总量和平均生成速度 |
| **当前轮** | 时间 | 该轮创建时间（精确到秒） |
| | 用时 | 该轮从发送到接收完成的总耗时 |
| | tok/s | 该轮的 token 生成速度 |
| | prompt/completion/total/rt | 该轮的各类 token 消耗 |
| | 输出字符/思考字符 | content 和 reasoning 的字数 |
| **轮次列表** | #轮次号 rt:N | 每轮序号和 reasoning_tokens 数，选中的高亮 |

### 右侧面板

| 区域 | 内容 |
|:-----|:------|
| **提示词** | 该轮的完整 prompt（system + user + 历史 assistant） |
| **思考过程** | AI 的 reasoning_content（原始思考文本） |
| **回答** | AI 的 content（最终回答文本） |

### 翻页

- `← 上一轮` / `下一轮 →` 逐轮翻看
- 直接点击左侧轮次列表跳转
- 标题显示 `第 X 轮（N / 总）`

## 五、模板

| 操作 | 位置 | 说明 |
|:-----|:------|:------|
| 保存当前为模板 | `📋 模板` → `💾 保存当前为模板` | 把当前积木列表保存为可复用的模板 |
| 插入模板 | `📋 模板` → 选择模板名 | 将已有模板的积木插入到当前列表末尾 |
| 删除模板 | `📋 模板` → `🗑 删除模板` | 弹窗列出所有模板，逐一点击删除 |

## 六、实验保存

- **手动保存**：点 `💾 保存`，生成 .md 文件到 experiments 目录
- **批量保存**：批量运行结束后自动保存每轮结果 + 自动生成 `_summary.md`
- **保存内容**：原始积木（含 reasoning 和 content）、API 请求体、API 响应体、usage 统计、stats（token/字符/语言/首行摘要）、实验备注
