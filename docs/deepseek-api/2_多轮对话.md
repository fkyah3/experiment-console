# 多轮对话

DeepSeek `/chat/completions` API 是一个**"无状态"API**——服务端不记录用户请求的上下文。用户在每次请求时，**需将之前所有对话历史拼接好后**传递给 API。

## 拼接方式

```python
from openai import OpenAI
client = OpenAI(api_key="<DeepSeek API Key>", base_url="https://api.deepseek.com")

# Round 1
messages = [{"role": "user", "content": "What's the highest mountain in the world?"}]
response = client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=messages
)
messages.append(response.choices[0].message)

# Round 2——追加上一轮的 assistant 回复 + 新提问
messages.append({"role": "user", "content": "What is the second?"})
response = client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=messages
)
messages.append(response.choices[0].message)
```

第一轮请求的 `messages`：

```json
[
    {"role": "user", "content": "What's the highest mountain in the world?"}
]
```

第二轮请求的 `messages`：

```json
[
    {"role": "user", "content": "What's the highest mountain in the world?"},
    {"role": "assistant", "content": "Mount Everest"},
    {"role": "user", "content": "What is the second?"}
]
```

## 思考模式下的特殊处理

### 无工具调用

`reasoning_content` 无需回传，API 会自动忽略：

```python
# Turn 2——直接 append assistant message，reasoning_content 会被忽略
messages.append(response.choices[0].message)
messages.append({'role': 'user', 'content': "How many Rs are there in the word 'strawberry'?"})
```

### 有工具调用

**必须**完整回传 `reasoning_content`，否则 API 返回 400。

```python
# 正确做法：直接 append response.choices[0].message
# 它等价于：
messages.append({
    'role': 'assistant',
    'content': response.choices[0].message.content,
    'reasoning_content': response.choices[0].message.reasoning_content,
    'tool_calls': response.choices[0].message.tool_calls,
})
```
