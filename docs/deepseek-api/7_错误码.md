# 错误码

DeepSeek API 的完整错误码列表。

| 错误码 | 描述 | 原因 | 处理方式 |
|:------|:-----|:-----|:---------|
| **400** | 格式错误 | 请求体格式错误 | 根据错误信息修改请求体 |
| **401** | 认证失败 | API key 错误 | 检查 API key 是否正确，或重新创建 |
| **402** | 余额不足 | 账号余额不足 | 确认账户余额，前往充值页面充值 |
| **422** | 参数错误 | 请求体参数错误 | 根据错误信息修改相关参数 |
| **429** | 请求速率达到上限 | TPM 或 RPM 达到上限 | 合理规划请求速率，等待后重试 |
| **500** | 服务器故障 | 服务器内部故障 | 等待后重试，若持续存在请联系官方 |
| **503** | 服务器繁忙 | 服务器负载过高 | 稍后重试 |

## 重试策略建议

### 可重试（TRANSIENT）

| 错误码 | 重试策略 |
|:-------|:---------|
| 429 | 指数退避 + jitter，检查 Retry-After 头 |
| 500 | 指数退避 + jitter |
| 503 | 指数退避 + jitter |
| 网络超时/连接错误 | 立即重试 1 次，再指数退避 |

### 不可重试（PERSISTENT）

| 错误码 | 处理方式 |
|:-------|:---------|
| 400 | 快速失败，向用户报错 |
| 401 | 快速失败，提示检查 API key |
| 402 | 快速失败，提示充值 |
| 422 | 快速失败，提示参数错误 |

### 推荐参数

- 最大重试次数：3 次
- 初始退避：1 秒
- 最大退避：30 秒
- jitter：0-1 秒随机偏移
