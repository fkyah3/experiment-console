# 限速与流式行为

## 速率限制

DeepSeek API 会根据负载情况，动态限制用户并发量。当到达并发上限时，立即返回 HTTP 429。

## 请求排队与 Keep-Alive

请求发出后可能需要等待调度。在等待期间，HTTP 连接会保持打开：

| 请求模式 | 等待期间的返回 |
|:---------|:--------------|
| 非流式（stream=false） | 持续返回空行 |
| 流式（stream=true） | 持续返回 SSE keep-alive 注释：`: keep-alive` |

这些内容不影响对响应 JSON body 的解析。如果自己解析 HTTP 响应，**必须处理这些空行和注释**。

## 超时阈值

| 超时类型 | 阈值 |
|:---------|:-----|
| 请求排队超时 | 10 分钟（超过后服务器关闭连接） |

## 流式响应格式

```
data: {"choices":[{"index":0,"delta":{"content":"你好"},"finish_reason":null}]}
                                       ↑ chunk 之间用 \n\n 分隔
data: {"choices":[{"index":0,"delta":{"content":"世界"},"finish_reason":null}]}

data: [DONE]
```

### 思考模式下的 chunk

```json
// reasoning_content chunk
{"choices":[{"index":0,"delta":{"reasoning_content":"思考过程..."},"finish_reason":null}]}

// content chunk
{"choices":[{"index":0,"delta":{"content":"最终回答..."},"finish_reason":null}]}

// 结束
{"choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}

// 最后 chunk 含 usage
{"choices":[],"usage":{"prompt_tokens":100,"completion_tokens":50,"total_tokens":150}}

// 流式结束标记
data: [DONE]
```

### 客户端处理要点

1. 按 `\n\n` 分割事件
2. 忽略空行和 `: keep-alive` 注释行
3. 只处理 `data: ` 开头的行
4. 处理不完整 JSON（断流场景）
5. 遇到 `data: [DONE]` 结束
6. `delta.reasoning_content` 和 `delta.content` 不会同时出现，需分别累积
